# Brainstorm Rule

## Objective

Produce unattended task understanding and implementation risk notes before editing business code.

## Output File

`.loopforge/brainstorm/brainstorm.md`

## Required Sections

```markdown
# Brainstorm

## Task Understanding

## Key Assumptions

## Risk Areas

## Candidate Target Files

## Candidate Test Files

## Questions Resolved by Assumption

## Suggested Execution Scope
```

## Guardrail

The brainstorm phase must not modify business code.
