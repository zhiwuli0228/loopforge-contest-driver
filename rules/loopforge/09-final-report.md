# Final Report Rule

## Objective

Produce a readable evidence-based final report even when execution is degraded.

## Primary Output

`.loopforge/reports/final-report.md`

## Fallback Output

`LOOPFORGE_FINAL_REPORT.md`

## Required Sections

```markdown
# LoopForge Final Report

## Result

## Mode

## Project Type

## Summary

## Changed Files

## Verification Commands

## Verification Results

## Gate Events

## Degradation

## Risks

## Traceability
```

## Reporting Rule

Never claim success without verification evidence. If verification is unavailable in the current milestone, report that limitation explicitly.
